# Plan: Gene Metadata Extraction

## Goal

Build two model-derived reference tables to support RNAseq dataset failure analysis
in `sms_omics`:

1. **`gene_metadata.tsv`** — one row per gene, boolean columns describing the gene's
   role in the model and whether it carries manual adjustments.
2. **`ref_basal_abundances.tsv`** — one row per gene, steady-state mean mRNA and
   monomer counts (and expression fractions) for the reference basal condition from
   a completed parca run.

These tables are generated by a vEcoli extraction script and output into the vEcoli
`out/` directory. They should be manually copied into `sms_omics/data/` for use in
analysis notebooks.

---

## Artifact 1: `gene_metadata.tsv`

### Columns

| Column | Type | Source |
|---|---|---|
| `gene_id` | str | EcoCyc ID — primary key |
| `gene_symbol` | str | from `raw_data.genes` |
| `in_model` | bool | gene appears in `sim_data.process.transcription.cistron_data['id']` |
| `aa_pway_enzyme` | bool | gene product appears as enzyme in aa synthesis kcat fitting (from `sim_data.process.metabolism.aa_synthesis_pathways` or enzyme kinetics tables) |
| `is_tf` | bool | gene product is a TF — appears in `sim_data.tf_to_active_inactive_conditions` key set (mapped back through monomer→gene) |
| `is_ribosomal_translation_mach` | bool | ribosomal subunit, translation factor, or tRNA synthetase — extracted from sim_data ribosome/translation data structures |
| `is_rnap_transcription_mach` | bool | RNAP subunit or sigma factor — extracted from sim_data RNAP / sigma factor data |
| `has_expression_adjustment` | bool | listed in `flat/adjustments/rna_expression_adjustments.tsv` |
| `expression_adjustment_factor` | float | the adjustment value (NaN if absent) |
| `has_deg_rate_adjustment` | bool | listed in `flat/adjustments/rna_deg_rates_adjustments.tsv` |
| `deg_rate_adjustment_factor` | float | the adjustment value (NaN if absent) |

### Notes

- `is_essential` is **not** extracted here because it lives in `sms_omics/data/essential_genes.tsv`
  (not in vEcoli). Join it in sms_omics by `gene_id` after copying the file over.
- The boolean columns are non-exclusive — a gene can be `aa_pway_enzyme` and
  `has_expression_adjustment` simultaneously (metA / EG10581 is a concrete example).
- Genes not `in_model` will have NaN for all model-role columns; include them so the
  table covers the full EcoCyc gene universe and can serve as a join target.

## Artifact 2: `ref_basal_abundances.tsv`

### Columns

| Column | Type | Notes |
|---|---|---|
| `gene_id` | str | EcoCyc ID — primary key |
| `gene_symbol` | str | |
| `mrna_expression_fraction` | float | `sim_data.process.transcription.rna_expression['basal']` mapped from TU → cistron → gene; sum over TUs containing the cistron if polycistronicAA |
| `mrna_ss_mean_count` | float | fraction × total expected mRNA count at basal condition (derived from mass data) |
| `monomer_expression_fraction` | float | monomer expression fraction at basal condition |
| `monomer_ss_mean_count` | float | fraction × total expected monomer count |

### Notes

- `rna_expression['basal']` is per transcription unit (TU); must be mapped back to
  cistrons and then to genes. For polycistronic TUs the fraction is shared — document
  the mapping convention (e.g. assign equal share to each cistron, or use max).
- Total mRNA and monomer counts at basal condition are available from `sim_data.mass`
  and cell geometry; confirm exact attribute paths during implementation.
- These are parca-fitted values, not simulation output — they represent the model's
  *target* steady state for the reference condition, not a simulated trajectory mean.
  Label clearly in the output as `parca_basal_*` if ambiguity is a concern.

## Extraction Script

**Location:** `runscripts/extract_gene_metadata.py`

**Inputs:**
- `--sim_data_path` — path to a completed parca `simData.cPickle`
- `--output_dir` — directory to write TSVs (default: `out/gene_metadata/`)

**Outputs:** Both TSVs written to `--output_dir`.

**Dependencies:** Loads `simData.cPickle` via pickle + reads flat adjustment TSVs
directly from `reconstruction/ecoli/flat/adjustments/`. No Nextflow or workflow
integration needed — run standalone after any parca completes.

---

## Integration with sms_omics

1. Copy both TSVs to `sms_omics/data/`.
2. In notebook 3 or a new notebook 4a, join `gene_metadata.tsv` and
   `ref_basal_abundances.tsv` to the expression matrix (`data_all.h5ad.var`) by
   `gene_id`.
3. Use boolean columns to define gene subsets for targeted comparison:
   - kcat-enzyme genes across datasets (failure mode 1 analysis)
   - TF genes across datasets (failure mode 2 analysis)
   - Adjustment-flagged genes (are the boosts sufficient given new dataset values?)
4. Use `ref_basal_abundances` to visualize how new dataset TPMs translate to expected
   model counts relative to the reference — this is the primary tool for rationally
   evaluating whether a dataset's expression profile is compatible with the model's
   assumptions.

`rnaseq-dataset-failure-analysis.md`, because the gene metadata table is a
prerequisite for doing those analyses in a structured way in sms_omics.

   (metA should appear with `has_expression_adjustment=True`, factor=10).

3. Run kcat-gene TPM comparison (replaces ad-hoc extraction done previously).
4. Run TF-gene TPM comparison for P-solve failure datasets.
